## Executive Summary

### Что мы построили

Passkey-based wallet SDK для Avalanche L1-приложений. Пользователи аутентифицируются отпечатком пальца/FaceID вместо сид-фраз. Разработчики интегрируют за 5 строк кода. Газ спонсируется через Paymaster. Кросс-чейн синхронизация ключей через Avalanche ICM. Развёрнуто и работает на 3 чейнах сегодня.

**OneClick — не потребительский кошелёк. Это невидимая инфраструктура для разработчиков, чьи пользователи не должны знать, что они на блокчейне.** Наша точка входа — гейминг на Beam, где friction при настройке кошелька напрямую убивает удержание игроков.

### Что жюри должно считать верным, чтобы этот бизнес работал

1. Пользователи Beam/игр ненавидят friction кошелька, и это измеримо вредит активации.
2. Passkey-онбординг улучшает completion rates по сравнению с сид-фразами.
3. Разработчики будут платить за управляемую инфраструктуру онбординга, если она быстро интегрируется и надёжна.

## 1. Почему сейчас

Четыре тренда сходятся, создавая окно для OneClick:

**Консолидация рынка embedded wallet инфраструктуры.** Stripe приобрёл Privy, Fireblocks приобрёл Dynamic. Оставшиеся независимые решения ориентированы на Ethereum, оставляя Avalanche недообслуженным.

**Расширение экосистемы Avalanche L1.** Активные mainnet L1 растут, каждому нужна инфраструктура для онбординга кошельков. Beam становится флагманским игровым чейном.

**Passkeys стали нормой для массовых пользователей.** Apple, Google и крупные платформы сделали биометрическую аутентификацию ежедневной привычкой. Пользователи ожидают логин по отпечатку — не 12-словные сид-фразы.

**Avalanche-specific технологические enablers.** Обновление Granite добавило P256 precompile (RIP-7212), а ICM обеспечивает кросс-чейн обмен сообщениями — оба являются фундаментом нашей архитектуры.

Мы принципиальны: passkey-first онбординг для Avalanche-приложений. Мы можем добавить другие методы аутентификации позже, но не раньше, чем биометрический поток будет полностью валидирован с реальным партнёрским трафиком.

## 2. Проблема

У Avalanche растущая экосистема L1. Каждому L1 нужна инфраструктура кошельков для онбординга пользователей. Большинство dApp по-прежнему полагаются на MetaMask или аналогичные расширения-кошельки, требуя от пользователей управления сид-фразами, газ-токенами и переключения сетей.

Это создаёт измеримый friction:

- Настройка кошелька занимает 3–5 минут для новых пользователей (установка расширения, бекап сид-фразы, покупка газа, настройка сети)
- Отраслевые отчёты указывают на значительный отвал при онбординге кошелька, особенно для не-крипто-нативных пользователей
- Игровые студии сообщают, что сложность кошелька — основной барьер для удержания игроков

### Наша гипотеза

Замена онбординга с сид-фразой на биометрическую (passkey) аутентификацию может сократить создание кошелька до нескольких секунд и существенно улучшить показатели завершения онбординга. Это должно быть валидировано с живыми партнёрскими данными.

### OneClick vs Традиционный кошелёк

| | Традиционный (MetaMask) | OneClick |
|---|---|---|
| Время онбординга | 3–5 минут | секунды, не минуты |
| Действие пользователя | Установить, записать 12 слов, купить газ, добавить сеть | Коснуться отпечатком |
| Сид-фраза | Обязательна | Нет (passkey в secure enclave) |
| Управление газом | Пользователь покупает и платит | Paymaster спонсирует, пользователь не в курсе |
| Переключение сетей | Вручную | Автоматический роутинг |
| Для разработчиков | WalletConnect / вручную | npm install, 5 строк кода |

### Критическая UX-проблема: Пустой кошелёк после создания

Создать кошелёк недостаточно. Кошелёк с нулевым балансом и без направленного действия приводит к немедленному оттоку. Мы решаем это на трёх уровнях:

- **Демо-среда:** Pre-funded кошельки с тестнет-токенами на Fuji (тестовая сеть Avalanche) для end-to-end тестирования новыми пользователями
- **Flow первого действия:** Направленная последовательность: создать кошелёк → купить первое крипто (фиат-онрамп) → сделать первый перевод
- **Шаблоны онбординга партнёров:** Шаблоны по use-case (гейминг: получить первый NFT, финтех: внести первый USDC), чтобы у пользователей была немедленная причина совершить транзакцию
- **Сигналы доверия:** Верифицированные контракты на Sourcify/Snowtrace + полностью open-source MIT кодовая база

## 4. Конкурентное позиционирование

Мы позиционируемся против традиционного UX-friction кошелька, не против конкретных кошельков. Альтернатива нашего пользователя — не «использовать MetaMask», а «не использовать крипто вообще».

| Конкурент | Их фокус | Наша дифференциация |
|-----------|---------|---------------------|
| MetaMask / WalletConnect | Браузерный кошелёк общего назначения | Нет SDK для удаления сид-фраз из dApp-потока |
| Biconomy / ZeroDev | ERC-4337 на Ethereum | Нет Avalanche ICM, нет мульти-L1 топологии, тяжелее инфра |
| Privy (купл. Stripe) | Embedded auth → платежи | Не Avalanche-native, closed source, vendor lock-in |
| Core Wallet (Ava Labs) | Consumer кошелёк, social login | B2C-приложение для крипто-aware; мы — B2B2C инфра SDK |
| Abstract Global Wallet | Один L2 на Ethereum | Только Ethereum, недоступен на Avalanche |

### Взаимоотношения с Core Wallet

Core Wallet обслуживает крипто-aware пользователей, которым нужен consumer-кошелёк с social login (Google/Apple). OneClick обслуживает другой сегмент: разработчиков, которым нужно встроить невидимую инфраструктуру кошелька в свои приложения для пользователей, не знающих что такое блокчейн. Это комплементарные продукты. Core — B2C; OneClick — B2B2C инфраструктура.

### Наша защитность

- **First mover:** Passkey wallet SDK для Avalanche L1 не существует
- **Отношения в экосистеме:** Build Games community, сеть менторов, pipeline Codebase/Retro9000
- **Глубина интеграции:** По мере встраивания OneClick в онбординг, конфигурацию paymaster и роутинг чейнов, стоимость переключения растёт естественно через глубину интеграции — не через закрытый доступ
- **Тайминг:** Рынок консолидируется (Privy→Stripe, Dynamic→Fireblocks). Экосистема Avalanche L1 растёт. Окно открыто сейчас

*Мы не пытаемся стать универсальной wallet-компанией. Мы строим узкий, принципиальный слой онбординга для Avalanche-приложений, где friction кошелька напрямую вредит активации.*

### Юнит-экономика (Ранняя стадия — Гипотеза)

- Стоимость инфраструктуры: ~$30/мес (Railway релеер + мониторинг)
- Breakeven: 1 Starter-клиент ($49/мес) покрывает инфраструктуру
- CAC (founder-led): $150–500 эквивалент времени на интеграцию
- Целевая gross margin: >80% на SaaS-тирах
- LTV: Слишком рано оценивать. Используем таргет payback period (<3 месяца)

### Обещание по времени интеграции

- **Quickstart:** 15–30 минут от npm install до первого созданного кошелька в sandbox
- **Production-ready:** <1 дня для простых кейсов (игровой логин, базовый send/receive)
- **Полная интеграция:** 1–3 дня с Smart Route, конфигурацией Paymaster и кастомным UI

Это наш самый важный GTM-рычаг. Если интеграция быстрая и безболезненная, adoption следует.

У нас сильный технический тракшн (развёрнутые контракты, live релеер, опубликованный SDK, проходящие тесты). Что мы ещё не доказали — это рыночный тракшн. Этот раздел перечисляет что нужно валидировать далее.

|-----------|------------------|---------------|------|

## 12. Позиция безопасности

Безопасность критична для инфраструктуры кошельков. Вот наша текущая позиция:

| Слой | Подход | Статус |
|------|--------|--------|
| Хранение ключей | Non-custodial. Приватный ключ passkey остаётся в secure enclave устройства (TPM/Secure Enclave). Мы никогда не видим и не храним приватные ключи. | Live |
| Смарт-контракты | Верифицированы на Sourcify + Snowtrace (exact match). 39/39 тестов. Документированный security audit с 29 находками. | Live |
| Релеер | Зашифрованные env vars на Railway. Единая точка отказа признана. Митигация: auto-restart, реплика планируется Месяц 2. | Частично |
| On-chain верификация | P256 precompile (0x0100) для верификации passkey подписей. Solidity-фоллбэк для L1 без precompile. | Live |
| Путь обновления | Multisig + timelock среднесрочно. Минимизация trust assumptions долгосрочно. | Планируется |

Полный отчёт аудита: SECURITY_AUDIT.md в репозитории.

## 13. Риски и митигация

| Риск | Вероятность | Импакт | Митигация |
|------|-----------|--------|-----------|
| Ни один dApp не интегрирует | Средняя | Критический | Прямой аутрич к 16+ проектам. Если спрос слабый — сузить до Beam-гейминга и поставить opinionated шаблоны онбординга. |

| Компрометация ключа релеера | Низкая | Критический | Зашифрованные env vars. Multisig + timelock среднесрочно. Минимизация привилегий релеера долгосрочно. |
| P256 не на всех L1 | Средняя | Средний | Solidity-фоллбэк для P256-верификации. Работать с L1-командами для включения precompile. |

## 14. Product Roadmap

### Phase 1: Wallet SDK (Завершено)
- ✅ Passkey/биометрическое создание кошелька
- ✅ Мульти-чейн поддержка (Fuji, C-Chain, Beam)
- ✅ Smart Route (авто-свап + перевод)
- ✅ Paymaster (спонсирование газа)
- ✅ ICM кросс-чейн синк ключей
- ✅ 39/39 тестов, security audit задокументирован

### Phase 2: Активация и монетизация (Месяц 2–4)
- Guided first-action flows и шаблоны онбординга партнёров
- Dev analytics dashboard
- Hosted Relayer SaaS-тиры
- Фиат-онрамп интеграция (MoonPay/Transak) — когда партнёрский спрос валидирует необходимость
- **Сессионный игровой режим:** Преавторизованные игровые сессии, устраняющие прерывания транзакциями во время геймплея. Игрок даёт одно согласие в начале сессии — все внутриигровые действия (покупки предметов, получение наград, минт NFT) собираются в пакет транзакций и отправляются вместе. Никаких попапов посреди игры. Пакетные транзакции значительно снижают стоимость газа на каждое действие. Техническая основа: сессионные ключи с лимитами по времени/сумме + слой батчинга транзакций на стороне релеера.

### Phase 3: Расширение (Месяц 4–6)
- Сессионные ключи для программного доступа
- P2P-переводы между OneClick-кошельками
- Авто-бридж между чейнами (расширение Smart Route)

### Phase 3.5: Встроенный DeFi Earn (Месяц 5–8)

Интеграция DEX-протоколов (TraderJoe, Benqi) напрямую в интерфейс кошелька для earn-функционала. Пользователь управляет стейкингом и лендингом в одном месте, не разбираясь в сложных механиках DeFi.

**Модель монетизации:** Небольшая комиссия с earn-операций (0.1–0.5%), прозрачная для пользователя.

**Почему это работает для всех сторон:**
- **Для DEX/протоколов:** Новый канал привлечения ликвидности от Web2-пользователей, которые иначе не пришли бы
- **Для пользователя:** Простой earn в одном интерфейсе — без навигации по DeFi-протоколам, без понимания LP, APR, impermanent loss
- **Для OneClick:** Понятная модель монетизации через комиссию, масштабируемая с ростом TVL

### Долгосрочные возможности (8–12+ месяцев)
Будущие опциональности, не текущие обязательства:
- AI Agent SDK (когда экосистема ERC-8183 созреет)
- WaaS (wallet-as-a-service) enterprise-тир
- Дополнительная поддержка EVM-чейнов за пределами Avalanche
